## 📊 الملخص التنفيذي

### وصف النظام

نظام ERP متكامل وشامل لإدارة موارد المؤسسات الصغيرة والمتوسطة، مبني كتطبيق ويب تقدمي (PWA) يعمل بالكامل باللغة العربية مع واجهة RTL. يغطي النظام دورة العمل الكاملة من المبيعات والمشتريات إلى المحاسبة وإدارة الموارد البشرية، مع دعم كامل للعمل بدون إنترنت وإمكانيات تعدد المستأجرين (Multi-Tenant SaaS).

### 1. المصادقة (Authentication)

| الميزة | التفاصيل |
|--------|----------|
| **التسجيل** | بريد إلكتروني + كلمة مرور مع تأكيد البريد |

| **حماية كلمة المرور** | مؤشر قوة كلمة المرور |

### 2. نظام الأدوار والصلاحيات (RBAC)

```
┌─────────────────────────────────────────────────────────┐
│                    المستخدم (User)                       │
│                         │                               │
│              ┌──────────┼──────────┐                    │
│              ▼          ▼          ▼                    │
│         user_roles  custom_roles  role_section_permissions│
│         (الدور)   (أدوار مخصصة) (صلاحيات الأقسام)       │
│              │                     │                    │
│              ▼                     ▼                    │
│         role_limits          check_section_permission()  │
│      (حدود مالية)           (التحقق من الصلاحية)        │
└─────────────────────────────────────────────────────────┘
```

**الأدوار الأساسية:**

| الدور | الوصف | الصلاحيات |
|-------|-------|----------|
| `admin` | مدير النظام | كل الصلاحيات |
| `sales` | موظف مبيعات | العملاء، المبيعات، الفواتير |
| `accountant` | محاسب | المحاسبة، المالية، التقارير |
| `warehouse` | أمين مخزون | المنتجات، المخزون، المستودعات |
| `hr` | موارد بشرية | الموظفين، المهام |

### 3. Row Level Security (RLS)

| الجانب | التفاصيل |
|--------|----------|
| **عدد السياسات** | 120+ سياسة على 56 جدول |
| **العزل** | عزل بيانات المستأجرين (tenant_id) |
| **الوظائف الآمنة** | `SECURITY DEFINER` مع `search_path = 'public'` |
| **التحقق من الدور** | `has_role()` function |
| **التحقق من الصلاحية** | `check_section_permission()` function |
| **التحقق من الحدود** | `check_financial_limit()` function |

### 4. Edge Functions (وظائف الخادم)

| الوظيفة | الغرض | التحقق |
|---------|-------|--------|
| `validate-invoice` | التحقق من صحة الفاتورة | بيانات + مبالغ + عميل |
| `approve-invoice` | الموافقة على الفاتورة | صلاحيات + حدود مالية |
| `process-payment` | معالجة الدفعات | رصيد + تحديث متزامن |
| `approve-expense` | الموافقة على المصروفات | صلاحيات + حدود |
| `stock-movement` | حركات المخزون | رصيد + تحديث ذري |
| `create-journal` | إنشاء قيد يومية | توازن مدين/دائن |
| `verify-totp` | التحقق من رمز 2FA | مفتاح سري + وقت |
