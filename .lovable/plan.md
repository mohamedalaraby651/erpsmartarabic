## الخطة التنفيذية

### المرحلة 1: الهيكل الأساسي

#### 1.1 رأس الصفحة (`CustomerPageHeader`)
- Desktop: عنوان "إدارة العملاء" + Breadcrumb (الرئيسية > العملاء) + عداد (25 من 150)
- أزرار: [إضافة عميل (Primary)] + [أدوات ▾ dropdown: تصدير متقدم، استيراد، كشف مكررين، دمج]
- Mobile: شريط بحث بارز في الأعلى + أيقونات (إضافة + أدوات) تحته

#### 1.2 شريط الفلاتر الذكية (`CustomerSmartFilters`)
- إعادة كتابة `CustomerStatsBar` كصف أزرار فلترة ذكية بأرقام:
  `[الكل 150] [نشط 120] [مدين 18] [VIP 12] [شركات 35] [مزارع 8]`
- الزر النشط بخلفية داكنة (bg-primary) والباقي outline
- على الموبايل: صف أفقي قابل للتمرير (ScrollArea)

#### 1.3 شريط البحث والفلاتر (`CustomerFiltersBar`)
- Desktop: صف واحد مدمج بدون Card محيطة — بحث + فلاتر dropdown + زر فلاتر متقدمة
- فلاتر متقدمة (في Popover أو Sheet):
  - فلترة الخمول (أيام بدون نشاط/تواصل)
  - نطاق الرصيد (من - إلى)
  - تاريخ الإضافة (من - إلى)
  - Saved Views: حفظ/تحميل مجموعات فلاتر مسماة (localStorage)

### المرحلة 2: عرض القائمة (Desktop)

#### 2.1 القائمة المدمجة (`CustomerListView`)
- كل عميل = سطر واحد بحد جانبي VIP ملون (border-s-4):
  - silver: zinc-400, gold: amber-500, platinum: purple-500, regular: transparent
- المحتوى: Avatar دائري + اسم + نوع badge صغير + VIP badge + هاتف + محافظة + رصيد ملون + نقطة حالة
- Hover: ظل خفيف + أيقونات إجراءات تظهر (فاتورة، دفعة، واتساب، تعديل، كشف حساب)
- النقر: انتقال لصفحة التفاصيل

#### 2.2 أعمدة قابلة للتخصيص
- زر ⚙️ بجانب الفلاتر يفتح Popover لاختيار الأعمدة الظاهرة
- أعمدة افتراضية: اسم، نوع، VIP، هاتف، محافظة، رصيد، حالة
- أعمدة إضافية: حد ائتمان، آخر نشاط، مشتريات، سداد%، تاريخ الإضافة
- الإعدادات تُحفظ في localStorage

#### 2.3 الترتيب
- النقر على عناوين الأعمدة للترتيب تصاعدي/تنازلي (سهم ↑↓)
- قائمة ترتيب منسدلة أيضاً للوصول السريع

### المرحلة 3: الموبايل

#### 3.1 بطاقة العميل (`CustomerMobileCard`)
- سطر بسيط مثل جهات الاتصال:
  - Avatar دائري + اسم + VIP badge صغير + رصيد ملون
  - حد جانبي VIP ملون
  - أيقونات إجراءات صغيرة ظاهرة (هاتف، واتساب، فاتورة)
- Swipe للإجراءات: سحب لليسار يكشف أزرار (تعديل، حذف، كشف حساب)
- النقر: انتقال لصفحة التفاصيل

#### 3.2 رأس الصفحة Mobile
- شريط بحث كبير في الأعلى (Search-first)
- تحته: أزرار فلترة ذكية (ScrollArea أفقي)
- زر إضافة + أدوات في الزاوية

### المرحلة 4: Infinite Scroll + Skeleton

#### 4.1 Infinite Scroll
- استبدال Pagination الحالي بـ IntersectionObserver
- تحميل 20 عميل في كل دفعة
- مؤشر تحميل في الأسفل عند جلب المزيد

#### 4.2 Skeleton Loading
- عند التحميل الأولي: عرض 8 skeleton rows بدل spinner
- كل skeleton row يحاكي شكل سطر العميل الحقيقي

### المرحلة 5: ميزات إضافية

#### 5.1 الحالة الفارغة (`CustomerEmptyState`)
- رسمة SVG متحركة (CSS animation — بدون Lottie لتجنب dependency ثقيلة)
- عنوان "لا يوجد عملاء بعد"
- خطوات إرشادية (1. أضف أول عميل 2. أنشئ فاتورة 3. تابع المبيعات)
- زر CTA "إضافة أول عميل"

#### 5.2 Dialog إضافة سريع
- حقول أساسية فقط: اسم، هاتف، نوع، محافظة
- رابط "إضافة متقدمة ←" يفتح الـ Wizard الكامل الموجود

#### 5.3 تصدير متقدم
- Dialog يحتوي: اختيار الأعمدة (checkboxes) + الصيغة (Excel/CSV/PDF) + نطاق البيانات (الكل/المصفى/نطاق تاريخ)

#### 5.4 Saved Views (فلاتر محفوظة)
- حفظ الفلاتر الحالية باسم مخصص في localStorage
- dropdown لتحميل أي مجموعة محفوظة
- إمكانية حذف المحفوظة

### المرحلة 6: تحسينات بصرية

- إزالة التحديد الجماعي (Checkbox, BulkActionsBar)
- إزالة التدرجات — ألوان flat محايدة
- Framer Motion: fade-in للقائمة، slide للفلاتر
- Dark Mode: مراجعة كل الألوان لضمان تباين ممتاز
- كثافة متوازنة: padding مناسب (p-3)، خطوط 13-14px
