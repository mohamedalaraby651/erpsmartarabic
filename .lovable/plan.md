## الخطة التنفيذية

### المرحلة 1: إعادة بناء `useCustomerAlerts` Hook

**الهدف**: توسيع أنواع التنبيهات من 3 إلى 8 وإضافة نظام أولويات.

**أنواع التنبيهات (مرتبة بالخطورة)**:
1. **تجاوز حد الائتمان** (error) — الحالي
2. **فواتير متأخرة 60+ يوم** (error) — الحالي
3. **اقتراب من حد الائتمان** (warning) — جديد: عند 80%+ من الحد
4. **دفعات مستحقة خلال 7 أيام** (warning) — جديد: فواتير ستستحق قريباً
5. **VIP بدون تواصل 30+ يوم** (warning) — جديد: عملاء VIP فقط
6. **انخفاض مبيعات** (info) — جديد: مقارنة آخر 30 يوم بالـ 30 السابقة
7. **عميل خامل 90+ يوم** (info) — الحالي
8. **عميل جديد بدون فواتير** (success) — الحالي

**تغييرات تقنية**:
- إضافة query لجلب فواتير مستحقة خلال 7 أيام
- إضافة query لمقارنة مبيعات شهرية (invoices aggregated by month)
- إضافة فلترة عملاء VIP بدون تواصل (last_communication_at)
- إضافة `priority` رقمي لكل تنبيه للترتيب
- إضافة `amount` اختياري لعرض القيمة المالية

### المرحلة 2: بانر التنبيهات الجديد (Desktop)

**`CustomerAlertsBanner.tsx`** — شريط badges ملونة أعلى القائمة:

```text
┌──────────────────────────────────────────────────────┐
│ ⚠ 3 تجاوز ائتمان  • ⏰ 5 متأخرة  • 📉 2 انخفاض   ⚙️ │
│                    [عرض التفاصيل ▾]                   │
├──────────────────────────────────────────────────────┤
│ (Accordion عند التوسيع)                               │
│ ┌─ تجاوز ائتمان ──────────────────────────────────┐ │
│ │ 🔴 أحمد محمود — 45,000 / 40,000 ج.م             │ │
│ │    [ملاحظة] [واتساب] [عرض العميل]               │ │
│ │ 🔴 محمد علي — 30,000 / 25,000 ج.م               │ │
│ │    [ملاحظة] [واتساب] [عرض العميل]               │ │
│ └──────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

**التصميم**:
- خلفية محايدة (`bg-muted/30`) مع أيقونات ملونة فقط
- كل badge = أيقونة ملونة + عدد + نص مختصر
- النقر على badge يُوسع Accordion لعرض تفاصيل هذا النوع فقط
- النقر على badge ثانية يُغلقه
- زر ⚙️ للانتقال لصفحة الإعدادات

**الإجراءات لكل تنبيه**:
- **ملاحظة**: Inline input يظهر تحت التنبيه مباشرة
- **واتساب**: `window.open(wa.me/phone)` بدون رسالة
- **عرض العميل**: `navigate(/customers/{id})`

### المرحلة 3: الموبايل — أيقونة + Bottom Sheet

**`CustomerAlertsMobileTrigger.tsx`**:
- أيقونة جرس عائمة بجانب البحث مع badge عدد التنبيهات
- النقر يفتح Bottom Sheet (Drawer from bottom)

**Bottom Sheet**:
- نفس محتوى الـ Accordion لكن في sheet
- مجموعة حسب النوع مع عناوين ملونة
- إجراءات swipe على كل تنبيه

### المرحلة 4: التفاعل مع القائمة

**فلترة بالنقر**: النقر على badge "تجاوز ائتمان" → القائمة تعرض فقط العملاء المتأثرين.

**Badge على كل عميل**: في `CustomerListRow` و `CustomerMobileCard`، عرض نقطة/رقم صغير بجانب اسم العميل إذا كان لديه تنبيهات نشطة.

**Highlight**: العملاء المتأثرون بتنبيهات error يحصلون على خلفية خفيفة `bg-destructive/5`.

### المرحلة 5: التسجيل التلقائي كإشعارات

عند حساب التنبيهات، مقارنة مع آخر تنبيهات مسجلة (localStorage). إذا ظهر تنبيه جديد لم يكن موجوداً:
- إدراج صف في جدول `notifications` بالعنوان والرسالة والنوع
- تشغيل صوت خفيف (`new Audio` مع ملف mp3 صغير)
- تحديث localStorage بالتنبيهات الحالية لتجنب التكرار

### المرحلة 6: صفحة إعدادات التنبيهات

**`/settings/alerts`** — صفحة منفصلة:

- **تشغيل/إيقاف كل نوع**: Toggle لكل نوع تنبيه
- **عتبات قابلة للتعديل**:
  - حد الائتمان: نسبة التحذير (افتراضي 80%)
  - الخمول: عدد الأيام (افتراضي 90)
  - VIP بدون تواصل: عدد الأيام (افتراضي 30)
  - الفواتير المتأخرة: عدد الأيام (افتراضي 60)
  - الدفعات القريبة: عدد الأيام (افتراضي 7)
  - انخفاض المبيعات: النسبة (افتراضي 30%)
- **فلترة بالأهمية**: اختيار الحد الأدنى للعرض (الكل / warning+ / error فقط)
- **الأصوات**: تشغيل/إيقاف

الإعدادات تُحفظ في `localStorage` تحت مفتاح `customer-alert-settings`.
