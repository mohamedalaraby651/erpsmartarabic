

# تقرير تحليلي شامل + خطة تطوير واجهة قسم العملاء

---

## القسم الأول: التحليل المقارن (الوضع الحالي vs المطلوب)

### 1. واجهة الموبايل — صفحة تفاصيل العميل

**الوضع الحالي:**
- `MobileStatsScroll`: شريط أفقي بسيط من 6 بطاقات صغيرة (110px) — أيقونة + رقم + تسمية فقط
- `MobileDetailSection`: 14 قسم عمودي بنظام Accordion بسيط — بدون تصنيف مرئي
- Hero Header مخفي على الموبايل (يظهر فقط `MobileDetailHeader` بسيط)

**المطلوب (استناداً للصورة المرجعية):**
- بروفايل مركزي علوي: صورة كبيرة دائرية + VIP Badge + الاسم + النوع + ID + الموقع + أزرار إجراء (فاتورة جديدة، كشف حساب)
- بطاقات إحصائية عمودية كاملة العرض: كل بطاقة بها أيقونة + عنوان + قيمة كبيرة + وصف فرعي + Progress (مثل حد الائتمان، نسبة السداد)
- تبويبات سفلية مبسطة: البيانات الأساسية، الفواتير، المدفوعات، التحليلات — بنظام Tab Bar أفقي
- قسم البيانات الأساسية: حقول مفصلة (الاسم، الهاتف، البريد، الرقم الضريبي، الفئة، تاريخ التسجيل) + مرفقات + تحليل المشتريات الشهرية

### 2. StatsGrid — ازدحام الشاشات المتوسطة

**الوضع الحالي:** `grid-cols-2 md:grid-cols-4 xl:grid-cols-8` — 8 بطاقات متساوية بدون أولوية

**المشكلة:** على شاشات md (768-1024px) تظهر 4 بطاقات/صف × صفين = مكتظ. لا يوجد تمييز بصري بين البطاقات الحرجة (الرصيد، المستحق) والثانوية (آخر شراء، DSO).

**المطلوب:** نظام أولوية — بطاقات رئيسية (الرصيد، المستحق، نسبة السداد) أكبر حجماً مع Progress bars + بطاقات ثانوية أصغر

### 3. تبويبات عروض الأسعار وأوامر البيع

**الوضع الحالي:** قائمة صماء بدون Summary Bar — فقط Badge + مبلغ + تاريخ

**المطلوب:** شريط ملخص (إجمالي المعلق، المقبول، المرفوض، معدل التحويل لفواتير)

### 4. سجل النشاط

**الوضع الحالي:** `activity.action` نص عادي + تاريخ — بدون أيقونات متنوعة أو تفاصيل التغيير

**المطلوب:** Timeline تفاعلي بأيقونات حسب نوع الحدث + اسم المستخدم + تفاصيل ما تغير

### 5. فلاتر Drawer المتقدمة

**الوضع الحالي:** فلاتر "بدون تواصل" و"بدون نشاط" في الـ Drawer فقط — لا تولد FilterChips

**المطلوب:** أي فلتر نشط يظهر كـ Chip في الشريط الرئيسي

## القسم الثاني: خطة التنفيذ التفصيلية

### المسار 1: إعادة تصميم واجهة الموبايل (الأولوية القصوى)

#### 1.1 إنشاء `CustomerMobileProfile.tsx` (مكون جديد)
- بروفايل مركزي: صورة دائرية كبيرة (96px) + ImageUpload overlay + VIP Badge فوق الصورة
- تحت الصورة: الاسم (xl bold) + نوع العميل + badge نشط/غير نشط
- معلومات مختصرة: ID + الموقع (المحافظة-المدينة)
- أزرار إجراء: "فاتورة جديدة" (primary full-width) + "كشف حساب" (outline full-width)

#### 1.2 إنشاء `CustomerMobileStatCard.tsx` (مكون جديد)
- بطاقة إحصائية عمودية كاملة العرض مع:
  - أيقونة ملونة (يسار) + عنوان + وصف فرعي
  - قيمة كبيرة (2xl bold) + وحدة (ر.س)
  - Progress bar اختياري (لحد الائتمان ونسبة السداد)
  - لون حدود يمين ديناميكي حسب الحالة (أخضر/أحمر/أصفر)

#### 1.3 تحديث `CustomerDetailsPage.tsx` — قسم الموبايل
- استبدال `MobileDetailHeader` + `MobileStatsScroll` + 14 `MobileDetailSection` بـ:
  1. `CustomerMobileProfile` (بروفايل علوي)
  2. 8 `CustomerMobileStatCard` (بطاقات عمودية — الرصيد، المشتريات، نسبة السداد، الفواتير، المدفوعات، DSO، المستحق، آخر شراء)
  3. Tab Bar أفقي ثابت أسفل البطاقات: (البيانات الأساسية | الفواتير | المدفوعات | المزيد)
  4. محتوى التبويب المختار يظهر أسفل الـ Tab Bar

### المسار 2: تحسين StatsGrid للشاشات المتوسطة

#### 2.1 تحديث `CustomerStatsGrid.tsx`
- تقسيم البطاقات إلى مستويين:
  - **رئيسية (3 بطاقات):** الرصيد + المستحق + نسبة السداد — تأخذ `col-span-2` على md مع Progress bars
  - **ثانوية (5 بطاقات):** المشتريات، المدفوعات، الفواتير، DSO، آخر شراء — حجم عادي
- Layout: `grid-cols-2 md:grid-cols-3 lg:grid-cols-4 xl:grid-cols-8` مع span ديناميكي

### المسار 3: ترقية تبويبات المبيعات

#### 3.1 تحديث `CustomerTabQuotations.tsx`
- إضافة Summary Bar: إجمالي العروض، المعلقة (amber)، المقبولة (emerald)، المرفوضة (destructive)
- إضافة Conversion Rate badge (عروض تحولت لفواتير)

#### 3.2 تحديث `CustomerTabOrders.tsx`
- إضافة Summary Bar: إجمالي الأوامر، المكتملة، المعلقة، إجمالي المبالغ

### المسار 4: ترقية سجل النشاط

#### 4.1 تحديث `CustomerTabActivity.tsx`
- تحويل من قائمة بسيطة إلى Timeline:
  - أيقونة ديناميكية حسب `action` (إنشاء → Plus، تعديل → Edit، حذف → Trash، دفعة → CreditCard)
  - عرض `changes` إذا متوفر (JSON diff)
  - عرض اسم المستخدم الذي قام بالحدث

### المسار 5: FilterChips للفلاتر المخفية

#### 5.1 تحديث `CustomerFiltersBar.tsx`
- إضافة chips لفلاتر "بدون تواصل منذ X يوم" و"بدون نشاط منذ X يوم" عندما تكون نشطة
- يتطلب تمرير `noCommDays` و `inactiveDays` كـ props إضافية

## القسم الرابع: ملاحظات التوافق

- جميع التعديلات تعتمد على نفس الـ Props الموجودة — لا حاجة لتغيير `useCustomerDetail` أو `customerRepository`
- المكونات الجديدة تستخدم نفس مكتبة UI (shadcn) والأنماط المعمارية (memo, lazy)
- لا تعديلات على قاعدة البيانات أو Edge Functions
- التصميم المرجعي (الصورة) يُستخدم كمرجع للأسلوب وليس للنسخ الحرفي
